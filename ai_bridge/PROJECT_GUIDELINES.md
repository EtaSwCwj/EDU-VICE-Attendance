## 🚨 절대 원칙

### 1. 말이 아닌 결과물로 검증

| ❌ 하지 말 것 | ✅ 해야 할 것 |
|--------------|--------------|
| "읽었어?" | "뭐라고 써있었어? 요약해봐" |
| "확인했어?" | "뭘 확인했는지 보여줘" |
| "이해했어?" | "이해한 거 요약해봐" |
| "인수인계 받아줘" | "인수인계 읽고 핵심 5가지 말해봐" |

**AI가 "읽었습니다", "확인했습니다" 하면 99% 안 읽은 거다.**

### 2. 인수인계 문서 필수 확인

새 채팅방 시작 시:
1. 인수인계 문서 **먼저** 읽기
2. 읽은 내용 **요약해서 보여주기**
3. 사용자 확인 후 작업 시작

**인수인계 안 읽고 작업하면 처음부터 다시.**

## 📝 지시서 작성 규칙 (선임 → 후임)

### 필수 이행 사항 (둘 다 해야 함)

```
1. filesystem:write_file → C:\github\ai_bridge\TASK_XXX_이름.md
2. present_files → 다운로드 링크 제공
```

**하나만 하면 안 됨. 둘 다 해야 함.**

### 파일명 규칙

| 종류 | 형식 | 예시 |
|------|------|------|
| 지시서 | `TASK_XXX_이름.md` (대문자) | `TASK_013_BACK_BUTTON.md` |
| 결과 | `task_XXX_result.md` (소문자) | `task_013_result.md` |

### 지시서 필수 포함 내용

```markdown
# TASK_XXX: 작업명

> **작성자**: 윈선임/맥선임 (메인 개발 n)
> **작성일**: YYYY-MM-DD
> **담당**: 윈후임/맥후임 (Sonnet)
> **결과 파일**: `C:\github\ai_bridge\task_XXX_result.md`
> **원칙**: 묻지 말고 끝까지 진행. 로그 필수. 앱 종료 = 테스트 끝.

## 📋 문제 상황
(왜 이 작업이 필요한지)

## 📋 작업 내용
(구체적인 코드 예시 포함)

## 📝 로그 확인 포인트
(어떤 로그가 나와야 정상인지)

## ✅ 완료 체크리스트
- [ ] 항목1
- [ ] 항목2

## 📝 결과 보고 템플릿
(후임이 채워야 할 양식)
```

### 후임 한계 (알고 있어야 함)

| 한계 | 대응 |
|------|------|
| 앱 종료 인지 못함 | 사용자가 "앱 꺼졌어" 말해야 함 |
| 타이핑 로그인 기록 못함 | 자동 로그인만 기록됨 |
| 테스트 중간에 멈춤 | 사용자 개입 필요 |

## 🎯 EDU-VICE 핵심 컨셉 (절대 잊지 말 것)

### 한 줄 요약
> "45p 2번 틀림 → 32p 복습" — 페이지+문제번호만으로 자동채점/분석

### 저작권 원칙

| ✅ 저장함 | ❌ 저장 안 함 |
|----------|--------------|
| 페이지 번호, 문제 번호 | 문제 원문 |
| 정답 (③), 단원명 | 지문, 해설 |
| 유형, 난이도 | 교재 이미지 |

**문제 원문/이미지 저장하면 안 됨 = 저작권 위반**

### 역할 체계 (5가지)

```
관리자 → 원장 초대, 전체 학원 관리
원장   → 선생/학생 초대, 학원 설정
선생   → 담당 학생 관리, 숙제 출제
학생   → 본인 데이터, 서포터 초대 (최대 2명)
서포터 → 조회 전용 (학부모)
```

### 복합 역할
- 한 유저 = 여러 컨텍스트 가능
- 예: A학원 선생 + B학원 학생 + 조카의 서포터
- 앱에서 컨텍스트 전환 UI 제공
